How to configure proxy setting for subclipse plugin?

1.     open file C:\Documents and Settings\<USER_ACCOUNT>\Application Data\Subversion\servers

2.     Modify the following proxy setting accordingly:

# http-proxy-host = defaultproxy.whatever.com

# http-proxy-port = 7000

# http-proxy-username = defaultusername

# http-proxy-password = defaultpassword

3.       Go to eclipse->window->preferences->Team->SVN, change SVN interface with SVNKit (Pure Java) enabled.

 

How to configure Maven global proxy setting?

1.     open <MAVEN_INSTALL_DIR>/conf/settings.xml;

2.     edit the following configuration accordingly:

    <proxies>

      <!-- proxy

       | Specification for one proxy, to be used in connecting to the network.

       | -->

      <proxy>

        <id>my-proxy</id>

        <active>true</active>

        <protocol>http</protocol>

        <username>proxyuser</username>

        <password>proxypass</password>

        <host>host</host>

        <port>port</port>

        <nonProxyHosts>local.net,some.host.com</nonProxyHosts>

      </proxy>

      <!-- -->

    </proxies>

How to use cargo plugin to deploy and run container for Maven web application projects?

1.     Open pom.xml of the project, and add informations below:

    <build>

      <plugins>

        <plugin>

           <groupId>org.codehaus.cargo</groupId>

           <artifactId>cargo-maven2-plugin</artifactId>

           <version>0.3-SNAPSHOT</version>

         </plugin>

      </plugins>

    </build>

    <pluginRepositories>

      <pluginRepository>

        <id>codehaus snapshot repository</id>

        <url>http://snapshots.repository.codehaus.org/</url>

        <releases>

          <enabled>true</enabled>

        </releases>

      </pluginRepository>

    </pluginRepositories>    

2.     Add goals to run configuration:

a)       Click "Open External Tools Dialog" in eclipse, then new a launch configuration.

b)       Set the "Base directory" with the project folder, and input "package cargo:start" to goals.  

3.     Debug the project:

a)       Select the "Debug" perspective in eclipse, and click the "run external tools" button

b)       With the configuration set, then debug information would display in the console.

How to generate the model class source from the .xsd file?

You can use JAXB to generate the model class source from the .xsd file and parse the xml file to java object automatically.

1.     Unzip the jaxb-ri.zip;

2.     Change directory to “bin” and execute command: xjc Cp [package] [xsd-file-path], the model class source would be generated in the current folder.

 

How to deploy Portal Application in Tomcat6x?

1.     Make sure Tomcat6x installed and started.

2.     Install Cyberelay Portal Maven plugin.  Under ${Base Directory}\portal-maven-plugin

        run mvn install -e -o      

3.     Using prefix:mojo  Invocation Syntax for Cyberelay Portal Maven plugin.

        Open <MAVEN_INSTALL_DIR>/conf/settings.xml, Added new PluginGroup as below:

        <settings>

          …

            <pluginGroups>

                 <pluginGroup>org.cyberelay.portal.plugin</pluginGroup>

             </pluginGroups>

          …

        </settings>

4.     Update Projects POM for tomcat6x deploy.

Open pom.xml of the project, and add informations below:    

        <plugin>

            <groupId>org.codehaus.mojo</groupId>

            <artifactId>tomcat-maven-plugin</artifactId>

            <configuration>

                <url>http://localhost:8080/manager/</url>

                <server>TomcatServer</server>

                <warFile>/target/portal.war</warFile>

            </configuration>

         </plugin>

       

      <plugin> 

           <artifactId>portal-maven-plugin</artifactId>

          <groupId>org.cyberelay.portal.plugin</groupId>

          <version>1.0.0</version>

      </plugin> 

5.     Deploy the Ceberelay Portal Application in Tomcat6x.

Click "Open External Tools Dialog" in eclipse, then new a launch configuration.

Set the "Base directory" with the project folder,

Input "clean package tomcat:deploy DeoployInTomcat6x:DeoployInTomcat6x" to goals.  

6.     Restart the Tomcat6x server. browser "http://localhost:8080/portal"
